{{- if .Values.devicePlugin.enabled }}

── FUSE device plugin (smarter-device-manager) ────────────────────────
The rio-device-plugin DaemonSet exposes /dev/fuse as the
`smarter-devices/fuse` extended resource. Builder pods request it via
resources.limits instead of a hostPath volume.

Prerequisites:
  • /dev/fuse present on every builder node (kernel FUSE module loaded).
    On most distros: `modprobe fuse` or ensure `fuse` is in modules-load.
  • Container runtime with idmap-mount support (containerd ≥ 1.7,
    CRI-O ≥ 1.25) for hostUsers:false to work. EKS/GKE managed
    nodegroups on AL2023/COS ship compatible runtimes by default.

Verify:
  kubectl get node -o json | jq '.items[].status.allocatable["smarter-devices/fuse"]'
  — should show {{ .Values.devicePlugin.fuseMaxDevices }} on builder nodes.
{{- end }}

{{- if and (not .Values.builderPoolDefaults.privileged) .Values.securityProfilesOperator.enabled }}

── Seccomp Localhost profile (security-profiles-operator) ─────────────
SeccompProfile CRs (rio-builder, rio-fetcher) are reconciled by SPO's
spod DaemonSet to /var/lib/kubelet/seccomp/operator/{name}.json.
Builder pods use seccompProfile: Localhost (RuntimeDefault + pivot_root
− ptrace/bpf/setns/process_vm_*).

Verify:
  kubectl get seccompprofile rio-builder rio-fetcher
  kubectl -n security-profiles-operator get ds spod

Builder pods on fresh nodes may briefly wait in the wait-seccomp init
container until spod reconciles the profile (seconds). See
docs/src/security.md r[worker.seccomp.localhost-profile].
{{- end }}

{{- if and .Values.controller.seccompPreinstalled .Values.karpenter.enabled }}

── Seccomp Localhost profile (Bottlerocket bootstrap, P0541) ──────────
Profiles are written to /var/lib/kubelet/seccomp/operator/ by the
rio-seccomp bootstrap container (EC2NodeClass userData, essential=true)
BEFORE kubelet starts — no spod DaemonSet, no wait-seccomp init.
Builder pods use seccompProfile: Localhost (RuntimeDefault + pivot_root
− ptrace/bpf/setns/process_vm_*).

Verify on a builder node:
  kubectl debug node/$(kubectl get node -l rio.build/node-role=builder -o name | head -1 | cut -d/ -f2) -it --image=busybox -- ls -l /host/var/lib/kubelet/seccomp/operator/

Profile updates: change files/seccomp-rio-*.json → next deploy bumps
the EC2NodeClass userData → Karpenter Drift rolls nodes. See
docs/src/security.md r[worker.seccomp.localhost-profile].
{{- end }}

{{- if .Values.builderPoolDefaults.privileged }}

── privileged:true escape hatch ───────────────────────────────────────
WARNING: builderPoolDefaults.privileged=true is active. This disables seccomp
entirely and is UNACCEPTABLE for multi-tenant deployments (ADR-012).
It is intended for k3s/kind clusters lacking the FUSE device plugin
or idmap-mount support. Production on EKS/GKE should use
privileged:false (the chart default).
{{- end }}
