Use `fileset.maybeMissing` so the build doesn't fail before the first `sqlx-prepare` run on a fresh clone (the directory won't exist yet; `query!` callsites gated behind `#[cfg(feature = "sqlx-checked")]` or similar won't be there yet either on first bootstrap ‚Äî but simpler: just run prepare once before merge).

Set `SQLX_OFFLINE = "true";` in `commonArgs` alongside `PROTOC` / `LIBCLANG_PATH`. This makes the macro read from `.sqlx/` instead of attempting a live DB connection at compile time.

### T2 ‚Äî `feat(tooling):` `just sqlx-prepare` target

MODIFY [`justfile`](../../justfile) ‚Äî add a recipe that spins up an ephemeral PG (via `rio-test-support::TestDb`), runs migrations, exports `DATABASE_URL`, and runs `cargo sqlx prepare --workspace`:

```just
# Regenerate .sqlx/ offline query cache. Required after any
# query!(...) macro edit or migration change. Spins up ephemeral
# PG (same bootstrap as TestDb), runs migrations, runs prepare.
sqlx-prepare:
    #!/usr/bin/env bash
    set -euo pipefail
    # TestDb-style ephemeral PG on a random port. Reuse the
    # scripts/pg-ephemeral.sh helper if it exists; else inline
    # the initdb + postgres + pg_isready dance here.
    PGDATA=$(mktemp -d)
    PGPORT=$(shuf -i 49152-65535 -n 1)
    trap 'pg_ctl -D "$PGDATA" stop -m immediate 2>/dev/null; rm -rf "$PGDATA"' EXIT
    initdb -D "$PGDATA" -A trust --no-sync >/dev/null
    pg_ctl -D "$PGDATA" -o "-p $PGPORT -k $PGDATA" start -w >/dev/null
    export DATABASE_URL="postgres://localhost:$PGPORT/postgres?host=$PGDATA"
    cargo sqlx migrate run --source migrations
    cargo sqlx prepare --workspace --check || cargo sqlx prepare --workspace
    echo "sqlx cache regenerated: $(ls .sqlx/ | wc -l) queries"
```

The `--check || prepare` pattern: `--check` exits non-zero if the cache is stale, in which case we regenerate. If it's already current, no-op.

Add `sqlx-cli` to `flake.nix` dev shell `nativeBuildInputs` (nixpkgs has it as `sqlx-cli`).

### T3 ‚Äî `feat(tooling):` pre-commit `.sqlx/` staleness check

Add a pre-commit hook entry (in `flake.nix` git-hooks config) that runs `cargo sqlx prepare --workspace --check` when any `*.rs` file containing `query!` is staged. Fast ‚Äî it doesn't spin up PG, just compares `.sqlx/*.json` hashes against the current macro callsites. Fails the commit if `.sqlx/` is stale ‚Üí developer runs `just sqlx-prepare` and re-stages.

Without this, `.sqlx/` will rot: someone changes a `query!` SQL string, forgets to regenerate, and CI fails 10 minutes later instead of at commit time.
