# Plan 506: xtask regen crds — serde_yml/PyYAML serialization mismatch breaks crds-drift

Discovered during [P0502](plan-0502-builderpool-sizing-enum.md) `.#ci`: `cargo xtask regen crds` produces output that FAILS the `crds-drift` check. P0502 merged by hand-running the PyYAML split. Any future CRD change hits this wall.

The mismatch is purely in YAML serialization style, not semantics. [`xtask/src/regen/crds.rs:32-41`](../../xtask/src/regen/crds.rs) parses crdgen's multi-doc output with `serde_yml::Deserializer::from_str`, extracts `metadata.name`, and writes each doc via `serde_yml::to_string(&v)`. [`flake.nix:1008-1016`](../../flake.nix) does the same split via PyYAML `yaml.dump(doc, sort_keys=False)`. serde_yml emits `description: |-` (block scalar literal) and quotes `jsonPath` values; PyYAML emits `description: "..."` (quoted plain scalar) and leaves `jsonPath` unquoted. `diff -r` at [`:1018`](../../flake.nix) sees every description field as different.

Compounding this: the drift-check error message at [`:1020`](../../flake.nix) says `Run: nix build .#crds && ./scripts/split-crds.sh result`. That script doesn't exist — `scripts/` is empty. Six stale references across the codebase all name the same deleted script.

## Tasks

### T1 — `fix(xtask):` extract shared split-crds.py, xtask calls it instead of serde_yml re-serialize

The drift check's PyYAML heredoc at [`flake.nix:1008-1017`](../../flake.nix) is the de-facto canonical serialization — the committed `infra/helm/crds/*.yaml` files were generated by it (via the deleted `split-crds.sh`, which wrapped the same python). Make xtask use the exact same path.

**Extract** [`flake.nix:1009-1016`](../../flake.nix) to NEW `scripts/split-crds.py`:

```python
#!/usr/bin/env python3
"""Split multi-doc crdgen YAML → one file per metadata.name.

Shared by `cargo xtask regen crds` and the crds-drift flake check.
Both callers diff-check byte-identity, so a single serialization
path (PyYAML `dump(sort_keys=False)`) is the only thing that makes
xtask output pass the drift gate.
"""
import sys, yaml, pathlib

src, out = sys.argv[1], pathlib.Path(sys.argv[2])
with open(src) as f:
    for doc in yaml.safe_load_all(f):
        if doc is None:
            continue
        name = doc["metadata"]["name"]
        (out / f"{name}.yaml").write_text(yaml.dump(doc, sort_keys=False))
```

**Rewrite** [`xtask/src/regen/crds.rs`](../../xtask/src/regen/crds.rs) `run()`:

Drop the `serde_yml::Deserializer` loop entirely. Write the raw `crdgen` stdout to a tempfile, then run `python3 scripts/split-crds.py <tmp> infra/helm/crds/`. The per-doc parse + `metadata.name` extraction + write all move to Python — xtask's only jobs are clearing `*.yaml`, running crdgen, and counting output files for the `ui::set_message`.

```rust
// After the ui::step("cargo run --bin crdgen", ...) that captures stdout:
let tmp = tempfile::NamedTempFile::new()?;
std::fs::write(tmp.path(), &yaml)?;

// Same python the drift check runs. serde_yml re-serialize (previous
// approach) produced `description: |-` where PyYAML emits quoted
// scalars; drift-check diff -r saw every description field as changed.
sh::run(cmd!(
    sh,
    "python3 scripts/split-crds.py {tmp_path} {out}"
))?;

let n = std::fs::read_dir(&out)?
    .filter_map(Result::ok)
    .filter(|e| e.path().extension().is_some_and(|x| x == "yaml"))
    .count();
ui::set_message(&format!("wrote {n} CRDs"));
```

Drop the now-unused `use serde::Deserialize` and the `serde_yml` dep from `xtask/Cargo.toml` if this was the only consumer (`grep serde_yml xtask/src/` at dispatch to confirm).

**Rewire** [`flake.nix:1008-1017`](../../flake.nix) `crds-drift` heredoc → call the script:

```nix
''
  mkdir -p $TMPDIR/split
  python3 ${./scripts/split-crds.py} ${crdsYaml} $TMPDIR/split
  diff -r $TMPDIR/split ${./infra/helm/crds} > $TMPDIR/diff || {
    echo "FAIL: crdgen output drifted from infra/helm/crds/" >&2
    echo "Run: cargo xtask regen crds" >&2
    cat $TMPDIR/diff >&2
    exit 1
  }
  touch $out
''
```

The `${./scripts/split-crds.py}` Nix path-interpolation copies the script into the derivation's input closure — no `nativeBuildInputs` change needed beyond the existing `py`.
