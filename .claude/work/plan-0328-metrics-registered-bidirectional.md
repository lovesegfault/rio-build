Bughunter finding. [`rio-scheduler/tests/metrics_registered.rs`](../../rio-scheduler/tests/metrics_registered.rs) is **unidirectional**: it proves every name in the hand-maintained `SCHEDULER_METRICS` list has a `describe_*!` call. It does NOT prove every `metrics::counter!()` emit-site has one. [P0214](plan-0214-per-build-timeout.md) added `metrics::counter!("rio_scheduler_build_timeouts_total")` at [`worker.rs:593`](../../rio-scheduler/src/actor/worker.rs) without a `describe_counter!` in [`lib.rs:49`](../../rio-scheduler/src/lib.rs) — and without adding to `SCHEDULER_METRICS`. CI stayed green. [P0212](plan-0212-gc-automation-cron.md) in the same window did 4/4 (emit + describe + test-list + observability.md). The test should have caught the disparity.

The comment at [`actor/tests/worker.rs:432-434`](../../rio-scheduler/src/actor/tests/worker.rs) **already knows** this gap: "`describe_counter!` and `counter!` take string literals independently". That test plugs it for one metric via a recorder that captures the actual increment — but it's per-metric manual coverage, not systematic.

**Emit→describe catches a real failure class.** A metric that's emitted-but-not-described appears in `/metrics` only on first increment, with no `# HELP` line. Grafana tooltips empty. Absence alerts can't distinguish "zero events" from "exporter not registered yet". The existing test's docstring at [`:1-8`](../../rio-scheduler/tests/metrics_registered.rs) describes exactly this — it just doesn't enforce it from the emit side.

/// Emit-side: every `metrics::{counter,gauge,histogram}!("name")`
/// literal in src/ has a `describe_*!` with the same name.
///
/// The spec-list→describe check above is unidirectional: a metric
/// added to code without touching SCHEDULER_METRICS (forgotten
/// step 3) AND without a describe_*! (forgotten step 2) sails
/// through. P0214's `rio_scheduler_build_timeouts_total` at
/// worker.rs:593 did exactly this — emit-only, CI green.
///
/// This check is necessarily a build-time source grep, not a
/// runtime recorder hook: you can't trigger every emit code path
/// from a unit test (most are in handlers gated on actor state).
/// The grep runs at test-compile-time via include_str! on a build
/// script output.
#[test]
fn all_emitted_metrics_are_described() {
    let emitted: Vec<&str> = EMITTED_METRICS.lines().filter(|l| !l.is_empty()).collect();
    // Precondition self-check: if the build-script grep returns zero
    // names, either src/ has no metrics (implausible — there are 37)
    // or the grep pattern broke. Fail loudly instead of passing vacuously.
    assert!(emitted.len() >= 30,
        "EMITTED_METRICS has only {} entries — build-script grep likely broke",
        emitted.len());

    let recorder = DescribedNames::default();
    let described = recorder.0.clone();
    metrics::with_local_recorder(&recorder, || {
        rio_scheduler::describe_metrics();
    });
    let described = described.lock().unwrap();

    let undesribed: Vec<_> = emitted.iter()
        .filter(|name| !described.iter().any(|d| d == *name))
        .collect();
    assert!(undesribed.is_empty(),
        "metrics emitted in src/ but NOT in describe_metrics():\n  {undesribed:#?}\n\
         \n\
         Add describe_counter!/describe_gauge!/describe_histogram! to \
         rio-scheduler/src/lib.rs:49 AND a row to docs/src/observability.md.");
}

// Generated by build.rs — see below.
const EMITTED_METRICS: &str = include_str!(concat!(env!("OUT_DIR"), "/emitted_metrics.txt"));
```

//! Test-build-script: grep metrics macro literals from src/.
//! Runs at `cargo build --tests` time. Output consumed via
//! include_str!(OUT_DIR/emitted_metrics.txt) in metrics_registered.rs.
use std::{env, fs, path::Path};

fn main() {
    let out = env::var("OUT_DIR").unwrap();
    let src = Path::new(env!("CARGO_MANIFEST_DIR")).join("src");
    let mut names = std::collections::BTreeSet::new();
    grep_metrics(&src, &mut names);
    fs::write(
        Path::new(&out).join("emitted_metrics.txt"),
        names.into_iter().collect::<Vec<_>>().join("\n"),
    ).unwrap();
    // Re-run if ANY src file changes. Coarse but correct — a new
    // metrics:: call anywhere in src/ invalidates the grep output.
    println!("cargo:rerun-if-changed=src");
}

fn grep_metrics(dir: &Path, out: &mut std::collections::BTreeSet<String>) {
    // Regex on raw .rs source. Matches metrics::counter!("lit"),
    // metrics::gauge!("lit"), metrics::histogram!("lit") — with or
    // without the metrics:: prefix (use-imported). Label args after
    // the name are ignored (", "label" => val).
    let re = regex::Regex::new(
        r#"(?:metrics::)?(?:counter|gauge|histogram)!\s*\(\s*"([a-z_]+)""#
    ).unwrap();
    for entry in fs::read_dir(dir).unwrap() {
        let path = entry.unwrap().path();
        if path.is_dir() {
            grep_metrics(&path, out);
        } else if path.extension().is_some_and(|e| e == "rs") {
            let text = fs::read_to_string(&path).unwrap();
            for cap in re.captures_iter(&text) {
                out.insert(cap[1].to_string());
            }
        }
    }
}
```
